# TaskFlow

Проектная система для строительных компаний. Планируйте, контролируйте, управляйте.

## Описание

TaskFlow - это веб-приложение на Vue.js 3 для управления строительными проектами, задачами, документами и календарем. Система предоставляет единую платформу для планирования, контроля этапов, управления ресурсами и документооборота.

## Технологии

- **Vue.js 3** - прогрессивный JavaScript фреймворк
- **Vite** - быстрый инструмент сборки
- **Vue Router** - маршрутизация для одностраничных приложений
- **CSS3** - стилизация и адаптивный дизайн

## Требования

- **Node.js** версии 16.0 или выше
- **npm** версии 7.0 или выше (или **yarn**, **pnpm**)

## Установка

1. Клонируйте репозиторий или скачайте проект:
```bash
git clone <repository-url>
cd StroykaMiigaik
```

2. Установите зависимости:
```bash
npm install
```

## Запуск проекта

### Режим разработки

Запустите сервер разработки с hot-reload:

```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:5173/`

### Сборка для production

Создайте оптимизированную сборку для production:

```bash
npm run build
```

Собранные файлы будут находиться в папке `dist/`.

### Предпросмотр production сборки

Для предпросмотра production сборки локально:

```bash
npm run preview
```

## Доступные страницы

- `/` - Главная страница

- `/reports` - Отчёты (требует авторизации)
- `/settings` - Настройки (требует авторизации)

## Разработка

### Добавление новых компонентов

Компоненты следует создавать в папке `src/components/` и импортировать там, где они используются.

### Добавление новых страниц

1. Создайте новый компонент в `src/views/`
2. Добавьте маршрут в `src/router/index.js`
3. При необходимости добавьте ссылку в навигацию (`src/components/Header.vue`)

### Стилизация

- Глобальные стили находятся в `src/assets/style.css`
- Стили для отдельных страниц находятся в `src/assets/styles/`
- Используйте CSS переменные для цветов и размеров (определены в `style.css`)

## Изображения

Изображения размещаются в папке `public/images/`. Для добавления новых изображений:

1. Поместите файл в соответствующую папку `public/images/`
2. Используйте путь `/images/filename.ext` в компонентах

**Примечание**: Если изображение отсутствует, приложение не сломается благодаря placeholder-файлам. Просто добавьте нужные изображения в соответствующие папки.
